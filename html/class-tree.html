Class Hierarchy

  • object: The most base type
    • StateMachines.sm.State: A state of a state machine.
      • StateMachines.decorator.state: This decorator class is used to declare the states of a state machine.
    • StateMachines.sm.Event: Base class for events processed by state machines (see StateMachine.process_event).
    • StateMachines.sm.Transition: A transition of a state machine.
      • StateMachines.decorator.transition: This decorator class is used to declare the transitions of a state machine.
    • exceptions.BaseException: Common base class for all exceptions
      • exceptions.Exception: Common base class for all non-exit exceptions.
        • StateMachines.decorator.TransitionDeclarationError: Signals that the @transition decorator must be used within a state declaration.
        • StateMachines.sm.StateAlreadyExists: Signals that a state by this name already exists.
        • StateMachines.sm.TransitionBadGuard: Signals that the guard specified for a transition is not callable.
        • StateMachines.sm.TransitionBadAction: Signals that the action specified for a transition is not callable.
        • StateMachines.decorator.TransitionBadActionName: Signals that the name of the function defining a transition must be action.
        • StateMachines.sm.StateUnknown: Signals that a state specified by name or by id is unknown.
    • StateMachines.sm.StateMachine: A basic state machine (see decorator.statemachine for the class that supports the @state and @transition decorators).
      • StateMachines.decorator.statemachine: A state machine that can use the @state and @transition decorators to declare the structure of the machine.
